My own private Sub-Pages? Edit

Latest comment: 13 December 20125 comments2 people in discussion

Can I make something like this? –Preceding unsigned comment was added by Jeansowaty (Talk|Contribs) 13:26, 2 December 2012 (UTC). Please sign your posts with ~~~~Reply[reply]

Sort of. The easiest way to do so is probably to edit your userpage and add a link with the name you want as a subpage of your userpage, and then to save your userpage, click the new link, and edit away. For instance, if you wanted to add a new "sandbox" subpage, you would edit your userpage, add the code [[User:Jeansowaty/sandbox]] somewhere on it, save, click the new link, and start editing. For other pages, just replace the "sandbox" in that code with the name of your choice. You should keep in mind, though, that these pages aren't private, in the sense that anyone will be able to see and edit them; there is no way to create a page that only you can see or edit. What is the maximum resolution for a GIF to be uploaded and still be animated? Edit

Latest comment: 11 December 20131 comment1 person in discussion

I have tried uploading a GIF several times. Even after trying to lower the quality, I still get this message:

Note: Due to technical limitations, thumbnails of high-resolution GIF images such as this one will not be animated.

The original GIF took up 2.12 MB, the second, after editing, only took up 1.15 MB.

What is the maximum file size for a GIF so that it can be animated and still be high resolution? –Preceding unsigned comment was added by 142.197.88.106 (talk) 03:31, 11 December 2013(UTC). Please sign your posts with ~~~~

The default limit is 12.5 million total pixels (including all frames), according to mediawiki. At 330x277 and 985 frames, File:Jeb Sheep.png uses slightly over 90 million pixels. Note that the limitation only applies when gifs are displayed as scaled thumbnails; when displayed at full size, it will always be animated. The reason for this limit is that the server has to do the work of resizing images when they're shown as thumbnails, and it's not very efficient when handling animated gifs; very large files could cause the server to hang or crash, so it simply refuses to resize them instead. -- Orthotopetalk 05:20, 11 December 2013 (UTC)Reply[reply]

How to have stacked references? Edit

Latest comment: 14 June 20173 comments2 people in discussion

I want to have a named note. I tried <ref name="name"; group="note> but that didn't work. Is that possible? – Nixinova • • 07:49, 13 June 2017 (UTC)Reply[reply]

You need to use <ref name="refName" group="refGroup">foo</ref>, followed by <references group="refGroup"/>. -- Orthotopetalk 08:43, 13 June 2017 (UTC)Reply[reply]
