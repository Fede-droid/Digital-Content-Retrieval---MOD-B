Template useless due to restrictions

Hi, i created a cool (imo) template, namely Template:BlockGrid, but it is currently unusable, since the loop limit (see below) is set to 100.

Many proper diagrams use more than 100 tiles, since images are two-dimensional (10×10 is not a very big area), and the limit is per page, so we would need several times the amount necessary for a single image.

An admin can configure the necessary parameter to allow as much looping as the software manages (which is far more than is needed for a single diagram) or a fixed value.

I would recommend setting the parameter to -1 (as mentioned here). I’m sure normal uses via the template wouldn’t stress the server too much (I would know, I’m a programmer). If somebody abuses it (by creating useless giant loops), he can be banned, but it would be very nice to be able to use my template more than ½ time ;)

To ease the decision: i could alter the template to circumvent this, since template argument loops are not affected by this, but that would involve a “|” after EVERY tile (as opposed to “after every row”) and use equal or more performance.

– Flying sheep 11:43, 12 April 2011 (UTC)Reply[reply]
