It's working properly again Reijerzzz (talk)
And the error is back again :( Reijerzzz (talk)
Works for me when looking at the doc page itself, but not when it's transcluded onto the template page. Are we hitting some kind of template usage or memory limit? -- Orthotopetalk 17:34, 23 April 2014 (UTC)
The script is right on the edge of the 7 second lua time usage limit. The problem is that I am manually splitting up and reading the lines of the IDs page, because I want to display the IDs in the order in which they are defined and use lua comments to define the headers. Lua tables have a completely arbitrary order, so it isn't possible to load it as a table (which is almost instantaneous). The splitting itself takes about 6 seconds (average of 0.1 seconds per line, in a 622 line file), the rest of the script takes about half a second. The only solution to this is to have another table which keeps the order the IDs should be read, however that would require manually updating that to keep it in sync with changes to the ID file, which this automatic documentation is trying to completely avoid. I do have solution to this, but I need to find the time to finish developing it. –Matt ᐸ Talk
Contribs ⎜ 00:29, 24 April 2014 (UTC)
