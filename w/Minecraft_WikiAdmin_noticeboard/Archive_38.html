Request for an anti-vandalism edit filter

This might be hard, but if a page starting with "Java Edition" is created, there should automatically be semi-protection for 24 - 48 hours (due to vandalism frequently appearing in new snapshot pages/new version pages) right after it is released/announced (like Java Edition 1.17, Java Edition 1.16.4, and Java Edition 21w15a). Humiebeetalk contribs 22:50, 14 May 2021 (UTC)

Creating the filter isn't hard, but auto-protection isn't available as an action. The filter would be something like:
(!user_name in page_recent_contributors) & (page_title irlike ^(Java Edition))
The actions available to a filter are: logging the filter hit, warning the user, throttling the activity that triggered the filter, disallowing the activity, revoking the user from being auto-promoted to various groups, blocking the user, removing the user from privileged groups, range-blocking an IP address range, or tagging the edit. Protecting the article doesn't seem to be available according to https://www.mediawiki.org/wiki/Extension:AbuseFilter/Actions . I'd say throttling and tagging would be the best options. Initial page creations can be tagged for review and if too many of them occur during a period of time, they are simply disallowed.
I'm not an edit filter manager here on this wiki so I can't tell if a new filter is needed or an existing one needs a modification. Amatulic (talk) 18:53, 16 May 2021 (UTC)
