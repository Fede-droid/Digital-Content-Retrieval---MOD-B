Add experimental toggles to update pages

Many of the newer updates have had experimental toggles, which are quite relevant when looking at the development versions of the updates. However, the semantics around all of this is quite confusing. Let's look at some examples.

  • Bedrock Edition 1.20.70 § Experimental has this leading paragraph:
    • These additions and changes are accessible by enabling the "Armadillo and Wolf Armor", "Update 1.21", and "Beta APIs" experimental toggles.
    • Note the lack of links.

I was thinking we could add some kind of a page for each experimental toggle, or just make it link to the respective update page. Update 1.21 already links to Tricky Trials, but this could be done for more toggles too. Considering how confusing this could all get though, it might be better to make a separate page for each experimental toggle. Another good idea would actually be to add a very long history section for each update that compiles all of the changes from all of the snapshots affecting the respective toggle. This would be useful for updates like 1.21, which has been spread across various development versions that are listed as dev versions for other updates, like 1.20.3 and 1.20.70.

It looks like experiments already has an okay history table on this, but I think it would be better to split things up be update. Also, I think we shoud links for Armadillo and Wolf Armor (to Armored Paws#Experimental Toggle) Next Major Update (to Trails & Tales#Experimental Toggle), Next Gameplay Update (to Caves & Cliffs#Experimental Toggle), Caves & Cliffs#Experimental Toggle. [#Experimental Toggle] could be renamed, or maybe we could make it the standard going forward?

I'm sorry if my writing style here is really sloppy, but really don't know how to write down any of these things. – Unsigned comment added by Simanelix (talk • contribs) at 07:30, 28 April 2024 (UTC). Sign comments with ~~~~Reply[reply]
