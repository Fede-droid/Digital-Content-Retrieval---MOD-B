Guide page texture changes

Currently, when a guide page indicates a texture change, it looks like this:

Name Image Changed
Gravel
  • Changed texture

But the problem with this is that it doesn't indicate how it looked before

Here is my suggestion:

Name Image Changed
Gravel
  • Changed texture from

I've already started incorporating this in my personal Legacy Console Edition guides, so it would be nice if everyone else decided to go along with it.

--MemphisM (talk) 17:38, 20 March 2024 (UTC)Reply[reply]

Personally I don't really like it. The amount of images would be overpowering the first-glance overview. Also it isn't really necessary anyway, the guides don't need to show all information about each facet of the update. - Jack McKalling (talk) 19:46, 20 March 2024 (UTC)Reply[reply]
I think showing the old image as a comparison might be nice, though it isn't the most necessary information and may be hard to pick out at that small size anyway. –Sonicwave talk 23:14, 25 March 2024 (UTC)Reply[reply]
Alternatively, what about showing some kind of small link in the "changed" column, that when hovered over, shows the same size previous version of the image in a hover effect? Or if we have the ability to do it, when clicked on, it could popup an image comparison display over a darkened page overlay (similar to what happens when you click on a gallery image), with the previous on the left and new on the right, potentially with zoomed-in scale. - Jack McKalling (talk) 09:17, 26 March 2024 (UTC)Reply[reply]
