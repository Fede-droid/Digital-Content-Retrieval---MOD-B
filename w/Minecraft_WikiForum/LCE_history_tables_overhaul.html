LCE history tables overhaul

Currently LCE Template:History tables don't specify what edition the version numbers in the columns refer to. Especially the PlayStation and Nintendo Switch version numbers are easy to mix up as they both use a similar format:

Legacy Console Edition
TU7CU1 1.0 Patch 11.0.1Added the enchanting system. 
{"type": "console", "xbox": "TU7", "xbone": "CU1", "ps": "1.0", "wiiu": "Patch 1", "switch": "1.0.1", "content": "Added the enchanting system."}
TU31CU19 1.22 Patch 3Enchanting now consumes lapis lazuli. 
{"type": "console", "xbox": "TU31", "xbone": "CU19", "ps": "1.22", "wiiu": "Patch 3", "switch": "", "content": "Enchanting now consumes (link to lapis lazuli article, displayed as lapis lazuli)."}
Enchanting has been re-balanced. 
{"type": "console", "xbox": "TU31", "xbone": "CU19", "ps": "1.22", "wiiu": "Patch 3", "switch": "", "content": "Enchanting has been re-balanced."}
Enchanted books can now receive multiple enchantments at once. 
{"type": "console", "version": "Enchanting now consumes [[lapis lazuli]].", "exp": "", "snap": "", "content": "(link to Enchanted book article, displayed as Enchanted books) can now receive multiple enchantments at once."}

Additionally, due to using the same version number, the PS3, PS4 and PSVita version numbers are currently combined in one column and link the general disambiguation page of the respective version number (that is sometimes shared with unrelated versions of other editions), even though separate pages for each version exist. I propose to introduce separate columns for PS3, PS4 and PSVita versions that link the respective pages directly.

As a solution for the first problem, I propose this modification of the LCE history table:

Legacy Console Edition
Xbox 360 Xbox One PS3 PSVita PS4 Wii U Switch  
TU7 CU1 1.00 1.00 1.00 Patch 1 1.0.1 Added the enchanting system.
TU31 CU19 1.22 1.22 1.22 Patch 3 Enchanting now consumes lapis lazuli.
Enchanting has been re-balanced.
Enchanted books can now receive multiple enchantments at once.

-- Delycache (Talk | Contributions) 11:54, 12 March 2024 (UTC)Reply[reply]
