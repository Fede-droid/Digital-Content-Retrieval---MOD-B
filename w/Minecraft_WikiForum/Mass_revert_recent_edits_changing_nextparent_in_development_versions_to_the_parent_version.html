Mass revert recent edits changing nextparent in development versions to the parent version

In the {{Version nav}} doc, this is how it's supposed to be used: 

|parent=
|prevparent=
|prev=
|next=
|nextparent=

It is very semantically clear about how this should be used. parent means the parent version. nextparent is the next parent version. User:StizzurpXDD has been mass changing nextparent in development versions to the parent version. These mass edits are made unilaterally – no discussion was involved. I acknowledge that it's a valid concern about how this might be confusing to some readers. However, if anything should have been changed, it is how the template deals with {{{parent}}}. nextparent should strictly mean the next parent version as it makes the most semantic sense to do so. Therefore, I suggest that these edits be reverted, and that changes regarding this problem should warrant a discussion before it is ever implemented.

Dianliang's proposal Edit

As Harristic pointed out, it's probably better to lead this discussion in a more productive way. Here's my proposal to solve it once and for all:

  • We get rid of nextparent for dev versions.
  • We use parent to take the place of nextparent in the nav when nextparent is left out.

This is good because:

  • Resolves the concern of the confusion
  • Make the most semantic sense
  • Relatively minimal changes

I'll repeat that the reason why I'm against the changes is not because I think the original is better for readers, but because how Stizzurp've implemented is just wrong:

“

even if we make the nav in chronological order, we should not use nextparent because it does not make semantic sense. Instead we should reuse the parent argument.

 „

I hope my solution make more sense. GIM Dianliang233 T C 10:37, 23 April 2024 (UTC)Reply[reply]

sii's proposal Edit

Change the version nav footer from this:

◄◄ prevparent ◄ prev next ► nextparent ►►

to this:

◄◄ prevparent parent nextparent ►►
◄ prev this next ►

Examples:

◄◄ 1.8 1.8.1 1.8.2 ►►
◄ 1.8.1-pre2 1.8.1-pre3 1.8.1-pre4 ►
◄◄ 1.15 1.16 1.17 ►►
◄ 1.16.2 1.16.3 1.16.4 ►
