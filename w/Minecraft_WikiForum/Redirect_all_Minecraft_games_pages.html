Redirect all Minecraft games pages

If a reader who is not familiar to how the Minecraft Wiki works can get a little lost finding Minecraft Legends, Dungeons and Story Mode pages because of the ':' that has to be put between the page name and the game name. So, to make it easier for the readers to navigate to these kind of pages, I suggest to redirect all pages of Minecraft Legends, Dungeons and Story Mode to their respective pages when writing the article name without the ':'.

For better understanding, here is an example: 'Minecraft Legends:Mobs' is the main page. Searching for 'Minecraft Legends Mobs' and 'Mobs Minecraft Legends' should redirect to the page 'Minecraft Legends:Mobs'

This should be done programmatically using Java Script instead of actually creating redirects one by one. -- StizzurpXDD(talk) 16:04, 13 April 2024 (UTC)Reply[reply]

Proposal #1 Edit

This is a JavaScript I have created that would redirect the Minecraft Legends, Minecraft Dungeons and Minecraft Earth pages from mainspace to their respective namespaces: 

// Check if the page URL contains "Minecraft Legends" and redirect if true
if (window.location.href.includes("Minecraft Legends")) {
window.location.href = window.location.href.replace("Minecraft Legends", "Minecraft Legends:");
}

// Check if the page URL contains "Minecraft Dungeons" and redirect if true
if (window.location.href.includes("Minecraft Dungeons")) {
window.location.href = window.location.href.replace("Minecraft Dungeons", "Minecraft Dungeons:");
}

// Check if the page URL contains "Minecraft Earth" and redirect if true
if (window.location.href.includes("Minecraft Earth")) {
window.location.href = window.location.href.replace("Minecraft Earth", "Minecraft Earth:");
}
Advantages
  • No new pages would be created for the redirect.
Disadvantages
  • It will not improve SEO (nothing will change).
  • It will not show articles in mainspace in Google Search Results (nothing will change).

Proposal #2 Edit

Instead of doing the redirects with the help of JavaScript or any other bot, it would be time-taking but a good idea with lesser disadvantages to create all the redirects manually. All the redirects would be created like any other redirect. Here is the code of what should be in the redirect page: 

#REDIRECT [[Minecraft (gamename):(articlename)]]
{{Redr|page in mainspace to a page in the Minecraft (gamename) namespace}}
Advantages
  • Will show up in Special:Search.
Disadvantages
  • Too many redirect category pages.

StizzurpXDD(talk) 17:13, 25 April 2024 (UTC)Reply[reply]

 Note There is already {{Redr|mcd}} for Dungeons redirects; similar redirect reasons for legends and earth can be added easily. | violine1101 (talk) 17:23, 25 April 2024 (UTC)Reply[reply]
The above was just an example. The main proposal is to create all the redirects manually. StizzurpXDD(talk) 09:51, 26 April 2024 (UTC)Reply[reply]

Proposal #3 Edit

Based on some old script of mine, adding a namespace filter when starting to search: 

//$('.ns-0 #simpleSearch #searchInput').click( ':', searchNamespace );
$('.ns-10000 #simpleSearch #searchInput').click( 'Dungeons:', searchNamespace );
$('.ns-10002 #simpleSearch #searchInput').click( 'Earth:', searchNamespace );
$('.ns-10004 #simpleSearch #searchInput').click( 'Legends:', searchNamespace );
$('.ns-10006 #simpleSearch #searchInput').click( 'Story Mode:', searchNamespace );
function searchNamespace(namespace) {
	$(this).val(namespace.data+$(this).val());
	$(this).off('click');
}

This might be slightly confusing for user though, so only a  Neutral from myself. -- MarkusRost (talk) 17:22, 25 April 2024 (UTC)Reply[reply]
