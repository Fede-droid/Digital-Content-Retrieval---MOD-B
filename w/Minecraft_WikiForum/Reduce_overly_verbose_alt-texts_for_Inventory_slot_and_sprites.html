Reduce overly verbose alt-texts for Inventory slot and sprites

The so-called “Operation Sprite Ender” led to usage of individual sprite images instead of big spritesheets; the latter were used via CSS styles, whereas the former are added like regular images, allowing to use alt-texts to improve accessibility for cases where images themselves cannot be shown (something not easily possible when using background-image). However, these alt-texts appear to be grossly overdone. They mention the file’s name first, then have the text “Sprite image for <something> in Minecraft”, and if there is a link, mention the link’s target (“linking to <article>”). The first and the last part are not really related to what exactly the image portrays, and the rest is overly verbose and could be reduced to simply <something>.

In a text-only browser such as Links, those big alt-texts cause a huge mess:

  • “Invicon Wooden Pickaxe.png: Sprite image for Wooden Pickaxe in Minecraft linking to Wooden Pickaxe” should be simplified to just “Wooden Pickaxe”.

  • Here, most of the alt-texts are not needed as the sprites act as decoration for the following text. Only ItemSprite wooden-pickaxe.png needs an alt-text, and in this case, it should be simply “Pickaxe”.

  • In recipe UI displays, alt-texts make them completely unreadable. And this is only {{Smelting}}; with {{Crafting}} it is much worse. In this case, the tooltip’s titles (more specifically the “plain title”) should be used as alt-text.

Note that MediaWiki automatically adds alt-texts when a title is provided, rendering explicit usage of alt= unnecessary in many cases. When sprites are paired with text, which is usually the case for links and {{Breaking row}}, alt-text is probably not needed at all as the sprite only acts as a decoration for the following text; here an empty alt-text should be used to signal that.

What are your thoughts? — BabylonAS 07:27, 24 February 2024 (UTC)Reply[reply]

Actually, now that I’ve just noticed, the same issue applies to infobox images. Ideally, there should be a textual description of what is portrayed on each infobox image — if that’s not possible, I see little sense in them having any form of alt-text. And I doubt the infobox inventory slots immediately below them need it either. — BabylonAS 07:38, 24 February 2024 (UTC)Reply[reply]
 Support Simplifying it. It just doesn't make sense to mention the same thing 4 times in one sentence and when it comes text-based browsers, they end up creating alt nightmare. --TreeIsLife (talk) 08:24, 24 February 2024 (UTC)Reply[reply]
