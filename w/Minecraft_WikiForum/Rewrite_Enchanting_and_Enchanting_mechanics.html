Rewrite Enchanting and Enchanting mechanics

In my opinion Enchanting and Enchanting mechanics are both a bit messy. So, I plan to rewrite them both.

My plans are to:

  • Move [Enchanting] to [Enchantment].
  • Add an [§ Obtaining] section to [Enchantment].
  • Add a [§ Usage] section to [Enchatment], which will be what [Enchanting § Summary of enchantments] currently is.
    • [Enchanting § Summary of enchantments by item] will become [§ By item], a subsection of [§ Usage].
  • Move [Enchanting mechanics] to [Enchanting table mechanics].
  • See § Enchanting Move for more detailed plans.

Also, I would more improve information on the algorithm that the game uses for randomly selecting enchantments, so I can make a chart on how frequently each enchantment occurs.

I am going to write the 2 new versions of the pages on my user pages first:

  • User:Simanelix/Enchanting table mechanics
    • The proposalis for this to be copy-pasted into Enchanting table mechanics when I'm done writing it.
      • This will also replace the existing Enchanting mechanics page.
    • Page contents:
      • How the enchantment table decides the level to apply to an item when enchanting it.
      • How enchantments are applied to an item, given a certain number of levels to apply.
  • User:Simanelix/Enchantment
    • The proposalis for this to be copy-pasted into Enchantment when I'm done writing it.
      • This will also replace the existing Enchanting page.
    • Page contents:
      • Where to find each enchantment.
      • What each enchantment does.

If you want to help out, feel free to go to my user pages and edit them.

This proposal is mostly a list of plans, but I won't be publishing these edits in the main namespace until I get consensus. The same applies to the redirect changes that I will be doing: I need consensus before I can do any of these steps. I also need suggestions on how to do each individual part of this proposal before I can publish the edits.

  • I'm hoping to get the majority of the work on my user pages done over the course of the next, regardless of how many suggestions or how much feedback I get.

--Simanelix (T|C) 18:26, 4 March 2024 (UTC)Reply[reply]

Enchanting Move Edit

The sections of Enchanting clearly correlate with the sections of an item article.

Sections that should be renamed:

  1. [§ Enchanting equipment] -> [§ Obtaining: Enchanting table], [§ Usage: Anvil], [§ Usage: Grindstone]; this list would be more appropriate in Enchanting (disambiguation), if that page existed.
  2. [§ Enchanting methods] -> [§ Obtaining]. I will expand this section greatly. [Trading] doesn't need a section because its article describes all there is to describe.
    1. [§ Enchanting table]
      1. [§ Affecting offered enchantments] - this should be moved to [Enchanting table mechanics]
    2. [§ Generated Loot]
    3. [§ Mob Loot]
    4. [§ Fishing] - this links to Fishing as the main article. There is also a table describing the rate of each enchantment while fishing, including the rate of fishing-rod-related enchantments.
    5. [§ Anvil combinations] -> [§ Usage: Anvil]
    6. [§ Enchanted books] -> [§ Usage: Anvil]
      1. [§ Best Enchanting order] - replace with a see also to [Anvil mechanics]
  3. [§ Disenchanting] -> [§ Usage: Grindstone]
  4. [§ Summary of enchantment] -> [§ Usage: Summary]
  5. [§ Summary of enchantments by item] -> [§ Usage: Summary: By item]
  6. [§ Maximum effective values for enchantments] -> [§ Usage: Summary: Maximum effective levels]
    • This section is probably exclusive to Java Edition.
    • Needs a paragraph clarifying that max effective levels are only relevant when using commands or when modifying NBT.
    • Needs a paragraph clarifying the difference between an enchanting level and an experience level.

Sections that are mostly about enchantments, as components-of or modifications-to items:

  1. [§ Item data] - the [Enchantment] equivalent of [§ ID]
  2. [§ Sounds] - missing sounds for grindstone and anvil
  3. [§ Videos]
  4. [§ History]
  5. [§ Issues]
  6. [§ Gallery] - images related to both enchantments and enchanting tables
    1. [§ Screenshots]
    2. [§ Development images]
  7. [§ See also] - this is too empty. That's an easy fix.
  8. [§ References]
