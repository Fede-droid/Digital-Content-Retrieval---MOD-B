Proposed page move Edit

Latest comment: 2 November 20186 comments5 people in discussion

Capopanzone has proposed the page be moved to Player for the following reason:
No other entity pages use "The" in their title, and "Player" is the in-game name for this entity (see string entity.minecraft.player)

 Oppose as it is referred to as the player in the End Poem. -BDJP (t|c) 21:01, 24 October 2018 (UTC)Reply[reply]
If it was part of the name, "the" would be capitalized. the entity is called "Player", since the end poem is directed at one player (the one who sees it), that player is referred to as "the" player.
The actual entity name (entity.minecraft.player) should also be preferred over usage in big strings of text. (In case it isn't clear, I actually  Support moving.) FVbico (talk) 21:10, 24 October 2018 (UTC)Reply[reply]
As FVbico pointed out, "the player" in that context is a generic term used inside a text, so obviously it receives articles and stuff like that. The actual entity name for the player entity is just "Player", and you can check that by searching for the string entity.minecraft.player in the en_us.json file in any version from 18w02a onwards. Using the article just because a sentence directed to the actual player uses "the" before "player" would be like saying that the Ender Dragon page should be named "The Ender Dragon" because the the description for "The End... Again" advancement uses "the Ender Dragon".--Capopanzone (talk) 19:26, 25 October 2018 (UTC)Reply[reply]
 Support For above reasons. It's always been silly that the player had a definite article in its page title. The player redirect page doesn't have any history besides its creation, so a move would be easy. – Jack McKalling [ ] 19:53, 25 October 2018 (UTC)Reply[reply]
 Support Is inline with similar moves that have happened. –Majr ᐸ Talk
Contribs 05:16, 26 October 2018 (UTC)Reply[reply]
It's been a week, so moved (also an excuse to test move tagging). –Majr ᐸ Talk
Contribs 01:30, 2 November 2018 (UTC)Reply[reply]
